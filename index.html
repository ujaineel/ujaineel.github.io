Linux Interprocess Communication System

This project involves development of inter-process communicating system calls. What these system calls do, is allow processes/threads to communicate with each other through indirect communication with kernel mailbox objects. A process can send a message into the mailbox while the other process can receive and read the message. It is synchronized as to avoid race conditions or any other data corruption/manipulation.

Debugging logical errors in kernel-space development was the most difficult challenge I encountered. It was extremely difficult to identify logical or memory errors when working with kernel code, especially with system calls.

I like this project as I learned how Linux system calls work behind the scene and how to write them.

Linux Development With C

ASL Recognition

Whilst watching a YouTube video without any provided subtitles, I realized that while I can understand and infer what is said due to the language and sound, if I had to ever encounter someone who relied on the American Sign Language, then that would make communication difficult. Hence, my inspiration to attempt this project.

I utilized the TensorFlow library for its implementation of Neural Network and created a Convolutional Neural Network by using the Convolutional and Pooling layers along with hidden Dense layers and Batch Normalization. The training and testing datasets for the model were created from ASL image data from Kaggle. An attempt is made to translate basic ASL letters and few gestures by video capturing through OpenCV and sending the frames as input for the model prediction.

Currently, converting image recognition to object detection with conversion from CNN to YOLO.
