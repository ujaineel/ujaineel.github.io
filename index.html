Linux Interprocess Communication System

This project involves development of inter-process communicating system calls. What these system calls do, is allow processes/threads to communicate with each other through indirect communication with kernel mailbox objects. A process can send a message into the mailbox while the other process can receive and read the message. It is synchronized as to avoid race conditions or any other data corruption/manipulation.

Debugging logical errors in kernel-space development was the most difficult challenge I encountered. It was extremely difficult to identify logical or memory errors when working with kernel code, especially with system calls.

I like this project as I learned how Linux system calls work behind the scene and how to write them.

Linux Development With C
